Images are both an SEO opportunity and an SEO liability. Done right, they drive traffic from Google Image Search, support page relevance signals, and improve Core Web Vitals scores. Done wrong, they bloat page load times, send garbage signals to crawlers, and miss the organic visibility that comes from image search.

This is a complete, actionable guide covering every layer of image SEO — from filenames and alt text to structured data, lazy loading, and the page speed impact that most SEO practitioners underestimate.

## Why Image SEO Matters More Than Ever in 2026

Google Images accounts for roughly 22.6% of all web searches, according to SparkToro's 2023 zero-click search study (the most recent comprehensive dataset). That's a significant fraction of search volume that only surfaces results with properly optimized images.

Beyond image search, images directly affect two of Google's three Core Web Vitals:

**Largest Contentful Paint (LCP):** The LCP element is the hero image or main visual on most pages. Google's own data shows that pages with LCP under 2.5 seconds convert significantly better and rank better than those above 4 seconds. An uncompressed 2 MB hero image on a mobile connection can add 3–5 seconds to LCP by itself.

**Cumulative Layout Shift (CLS):** Images without explicit `width` and `height` attributes cause layout shifts as they load, pushing content around the page. Every layout shift contributes to a worse CLS score and a worse user experience.

Google confirmed Core Web Vitals as ranking signals in May 2021. Image optimization is now directly tied to search rankings, not just user experience.

## 1. Filenames: The Most Neglected SEO Signal

The filename of an image is the first signal a search engine has about the image's content. It's also the signal most consistently neglected in real-world workflows.

**Bad filenames (what most sites actually have):**
- `IMG_20241203_142857.jpg`
- `DSC_0047.png`
- `screenshot-2024-11-15.png`
- `image1.jpg`, `image2.jpg`, `image3.jpg`

**Good filenames (what search engines understand):**
- `espresso-machine-delonghi-dedica-review.jpg`
- `react-useeffect-cleanup-function-example.png`
- `hiking-boots-gore-tex-waterproof-review.webp`

The rules for SEO-optimized filenames:
- Use lowercase letters and hyphens to separate words (not underscores, not spaces)
- Describe the image content specifically and accurately
- Include the target keyword for the page naturally, not forcefully
- Keep it under 5–6 words — keyword stuffing filenames provides no benefit and looks spammy
- Match the filename to what's actually in the image

## 2. Alt Text: For Accessibility and SEO

Alt text (the `alt` attribute on `<img>` tags) serves two purposes: it describes the image to screen readers for visually impaired users, and it tells search engine crawlers what the image depicts when they can't render it.

```html
<!-- Bad: empty alt text -->
<img src="coffee-shop.jpg" alt="">

<!-- Bad: keyword stuffing -->
<img src="coffee-shop.jpg" alt="coffee shop best coffee shop near me coffee shop nyc">

<!-- Good: descriptive, accurate, includes relevant keyword naturally -->
<img src="barista-making-latte-art-independent-coffee.jpg"
     alt="Barista creating leaf pattern latte art in an independent coffee shop">
```

**Alt text best practices:**
- Describe what's actually in the image, not what you wish were in the image
- Include the page's target keyword if it fits naturally — don't force it
- Keep alt text under 125 characters (the limit at which screen readers typically truncate)
- Don't start with "Image of..." or "Photo of..." — it's redundant
- For purely decorative images (dividers, spacers, design elements), use an empty alt attribute (`alt=""`) — this tells screen readers to skip it

**What not to do:**
- Leave alt text empty on informational images (accessibility failure and SEO miss)
- Copy the filename into the alt attribute (defeats the purpose)
- Write the same alt text for multiple images on the same page
- Use alt text as a hidden keyword-stuffing vector — Google has been penalizing this since the Panda updates

## 3. Image Captions

Captions are the text immediately below images in an article or page. They're consistently among the most-read elements on any web page — studies show that captions are read 300% more than body copy. Yet most sites either skip them or write generic descriptions.

From an SEO perspective, caption text is crawlable by search engines and contributes to the page's topical relevance signals. A caption that naturally includes a relevant keyword strengthens the page's relevance to that keyword without feeling forced — because captions are a natural place for context.

**Good caption approach:** Describe what the image shows and add context that the image alone can't convey. "A 2019 study found that baristas at independent coffee shops spend an average of 7 minutes per latte art session, significantly more than chain coffee shops." This is useful to the reader and semantically relevant to the page topic.

## 4. Structured Data for Images

For certain image types, adding structured data (Schema.org markup) can enable rich results in Google Search — prominent image carousels, recipe images, product images, and more.

**Relevant Schema types:**

**Product:** For e-commerce product images, `Product` schema with `image` property tells Google which image represents the product, improving eligibility for Google Shopping and product-rich results.

```json
{
  "@context": "https://schema.org",
  "@type": "Product",
  "name": "DeLonghi Dedica Espresso Machine",
  "image": "https://example.com/images/delonghi-dedica-espresso-machine.webp"
}
```

**Article:** For blog posts, `Article` schema with `image` property (1200×630px minimum for Google's guidelines) is required for articles to appear as rich results with large images in search.

**Recipe:** The `Recipe` schema with `image` property enables the image carousel in Google's recipe search results, which drives significant additional clicks.

**ImageObject:** For detailed image metadata, the `ImageObject` type allows specifying `contentUrl`, `description`, `license`, and `creditText` — useful for stock photography sites, press pages, and photo portfolios.

## 5. Page Speed and Core Web Vitals

Image optimization's effect on page speed is not just a user experience concern — it directly affects rankings through Core Web Vitals. Here's the complete image performance checklist.

### Compress Images to Web-Appropriate Sizes

This is the highest-leverage action. Converting images from JPEG to WebP and applying appropriate compression can reduce total page image weight by 50–70%. See the complete guide on [how to compress images for a website](/blog/compress-images-for-website).

### Use Responsive Images with `srcset`

The `srcset` attribute lets browsers choose the appropriate image size for their viewport and pixel density:

```html
<img
  srcset="hero-400.webp 400w, hero-800.webp 800w, hero-1400.webp 1400w"
  sizes="(max-width: 600px) 400px, (max-width: 1200px) 800px, 1400px"
  src="hero-1400.webp"
  alt="Hero image description"
  width="1400"
  height="600"
>
```

A mobile visitor sees a 400px image instead of a 1400px image — that's a 90%+ reduction in data for mobile users.

### Specify `width` and `height` Attributes

Always include `width` and `height` on `<img>` tags. The browser uses these to reserve space in the layout before the image loads, preventing layout shift (CLS). Modern CSS (`height: auto` in your stylesheet) ensures the image still scales responsively.

### Use `loading="lazy"` for Below-the-Fold Images

```html
<img src="product-thumbnail.webp" width="400" height="300"
     alt="Product thumbnail" loading="lazy">
```

Lazy loading defers fetching images until they're about to enter the viewport. This reduces the initial page load payload significantly. Don't use `loading="lazy"` on the LCP image (usually the hero) — you want that to load as fast as possible.

### Preload the LCP Image

For hero images that are the LCP element, add a `<link rel="preload">` tag in the `<head>`:

```html
<link rel="preload" as="image" href="/images/hero.webp" fetchpriority="high">
```

This tells the browser to prioritize fetching this image, which can meaningfully improve LCP scores.

## 6. CDN: Reduce Latency for Global Visitors

A CDN (Content Delivery Network) stores copies of your images on servers distributed geographically. A visitor in Tokyo gets images from a Tokyo server instead of your server in Virginia — potentially cutting image load times by 100–300ms for distant visitors.

For image SEO, CDN matters because:
- Faster image delivery improves LCP, which is a ranking signal
- CDN logs give you visibility into which images are served most, informing optimization priorities
- Many CDNs (Cloudflare, Fastly, Akamai) offer automatic WebP conversion and image resizing at the edge

Cloudflare's free tier provides CDN functionality for static assets including images. For most small to medium sites, it's the easiest performance upgrade available.

## 7. File Size as a Direct Ranking Factor

Google has confirmed that it uses page experience signals — including Core Web Vitals — as ranking factors. The relationship between image file sizes and rankings is therefore not theoretical: large uncompressed images slow down LCP, which directly degrades your Core Web Vitals score.

A useful rule of thumb: no single image on a web page should exceed 200 KB. Hero images should be under 150 KB. Thumbnails should be under 50 KB. If your images are larger than these targets, they're almost certainly hurting your LCP score.

You can audit your current image performance using Google's PageSpeed Insights, which specifically calls out oversized images with estimated savings. Lighthouse (built into Chrome DevTools) provides the same analysis locally.

## FAQ

**Do image filenames really affect SEO?**

Yes, but with caveats. Descriptive filenames are a signal to both Google Image Search indexing and the page's topical relevance. The effect is modest for any single image but meaningful across a site with hundreds of images, all contributing relevance signals. The bigger win from good filenames is often image search visibility — `espresso-machine-review.jpg` will rank for image searches where `DSC_0047.jpg` never will.

**What's more important for image SEO — alt text or file compression?**

They serve different purposes and both matter. Alt text affects image search visibility and accessibility. File compression affects page speed, Core Web Vitals, and therefore overall rankings. On a typical site, compression has higher leverage — a 2-second improvement in LCP affects every page, every user, every day. Alt text improvements are additive over time as you update individual images.

**Should every image have alt text?**

Informational images should always have descriptive alt text. Purely decorative images (dividers, background textures, decorative icons) should have empty alt text (`alt=""`) — this tells assistive technologies to skip them, which is the correct behavior. The distinction matters both for accessibility compliance and for not polluting search engine signals with irrelevant text.
