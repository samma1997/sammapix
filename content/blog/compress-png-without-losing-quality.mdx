PNG is the right format for some images and the wrong format for many others. Understanding which category your image falls into — and knowing the compression techniques available for each — is the difference between a 50 KB logo that looks sharp at any size and a 4 MB product photo that should have been a 150 KB WebP.

This guide covers the mechanics of PNG compression, the lossless and near-lossless techniques available, the transparency/alpha channel question, and when you should convert PNG to another format entirely.

## Why PNG Is Different from JPEG

PNG (Portable Network Graphics) uses exclusively lossless compression. When you save a JPEG at 85% quality, the encoder permanently discards image data you won't notice. When you save a PNG, the encoder reorganizes and compresses the pixel data without removing any of it — the decompressed image is a bit-for-bit copy of what you put in.

This is why you can save a PNG and open it, edit one pixel, save it again, open it, and the image is still perfect. Do the same with JPEG and each save/re-encode cycle degrades the image slightly.

The trade-off: lossless compression is significantly less efficient for photographic content. A photograph saved as PNG is typically 5–10× larger than the same photograph at JPEG 85% quality. For logos, screenshots, and interface images, the lossless quality is worth the overhead. For photographs, it almost never is.

## The Two Approaches to PNG Compression

There are two fundamentally different strategies for making PNG files smaller:

### 1. Lossless PNG Optimization

Standard PNG compression uses DEFLATE — the same algorithm used by ZIP files. The first level of optimization is simply ensuring the DEFLATE implementation is as efficient as possible. Many image editors (including Photoshop, GIMP, and most export pipelines) use a fast but not maximally efficient DEFLATE implementation. Recompressing with a better optimizer like `pngcrush`, `oxipng`, or `zopflipng` can achieve 10–30% smaller files with zero quality change.

Additionally, the PNG format stores a "filter" applied to each row of pixels before compression. Choosing the optimal filter for each row (rather than using the same filter for all rows) can improve compressibility. The best optimizers try multiple filter combinations and keep the result with the smallest output.

**Expected savings from lossless optimization: 10–30%**

### 2. Lossy PNG Compression (Quantization)

Here's where it gets interesting: you can apply lossy compression to a PNG without changing the file format. The technique is called color quantization — reducing the number of unique colors in the image.

A full-color PNG can contain up to 16.7 million unique colors (24-bit RGB). An indexed PNG uses a palette of up to 256 colors. For many images — especially logos, illustrations, and simple graphics — the visual difference between 16.7 million colors and 256 carefully chosen colors is imperceptible. The file size difference is significant: an indexed 8-bit PNG is roughly 60–80% smaller than its full-color equivalent.

The quantization algorithm selects the 256 colors that best represent the original image and maps every pixel to the nearest palette color. Good quantization algorithms (like pngquant, which TinyPNG uses) apply dithering — adding subtle noise patterns that simulate intermediate colors and make the color reduction invisible to the human eye.

**Expected savings from quantization: 40–80%**

This is why TinyPNG (which uses pngquant under the hood) achieves such impressive reductions for PNG files — it's applying lossy quantization while keeping the file in PNG format. The output is technically a lossy version of the original, but visually identical for the vast majority of images.

## When Quantization Works and When It Doesn't

Color quantization is highly effective for:
- Logos and brand marks with limited color palettes
- Icons and UI elements
- Illustrations and cartoons
- Screenshots of interfaces (typically use few unique colors)
- Simple diagrams and charts

Quantization is less effective (and may show visible artifacts) for:
- Photographs with complex color gradients
- Images with smooth, subtle color transitions (skin tones, sky gradients)
- Images with fine noise or grain
- Medical or scientific images where color accuracy matters

The tell-tale sign of over-aggressive quantization is color banding — you'll see visible stepped transitions between what should be smooth gradients.

## The Alpha Channel: Preserving Transparency

PNG's defining advantage over JPEG is full transparency support via the alpha channel. An 8-bit alpha channel allows each pixel to have 256 levels of transparency, from fully opaque to fully transparent. This enables smooth anti-aliased edges on logos and icons, partial transparency in design elements, and complex transparency effects.

When compressing PNG files, preserving the alpha channel is often non-negotiable. A logo dropped onto a colored background needs to composite cleanly — any artifacts in the alpha channel show up as colored halos or jagged edges.

**What to watch for when compressing PNGs with transparency:**

Quantization on images with transparency can produce unwanted effects near the edges of transparent areas. The quantizer may merge similar transparent and opaque colors into a single palette entry, creating visible edge artifacts. Good PNG compressors handle this by separating the alpha channel from the color channels during quantization.

**The WebP solution:** WebP supports full alpha channel transparency, and lossy WebP at 80% quality with transparency typically produces files 60–80% smaller than an equivalent PNG with no visible quality difference. If you're using a PNG specifically for web delivery and it has transparency, converting to WebP is almost always the better option. ## PNG vs JPEG: How to Decide

The format decision should be made before compression, not after. Here's the framework:

**Use PNG when:**
- The image has transparency (JPEG cannot store transparency)
- The image contains text and sharpness is critical (logos, screenshots, interface mockups)
- The image will be further edited (lossless means no generation loss)
- The image has large areas of solid color with sharp edges (icons, line art)
- Pixel-perfect reproduction is required

**Use JPEG when:**
- The image is a photograph with complex color gradients
- Transparency is not needed
- You need the smallest possible file size for photographic content

**Use WebP when:**
- You're delivering for the web and browser support (96%+ in 2026) is acceptable
- You want the best of both worlds: smaller than JPEG for photos, smaller than PNG for graphics with transparency
- You need animation without GIF's quality limitations

For web delivery in 2026, the practical answer is: start with WebP. If you have a specific reason to use PNG (transparency for a logo that a content editor needs to download and place in a presentation, for example), use PNG. For everything else on the web, WebP is smaller with equivalent quality.

## Common Mistakes When Compressing PNG Files

**Converting screenshots with text to JPEG.** JPEG's compression creates visible artifacts around high-contrast edges — exactly what text characters are. A screenshot of a code editor or an interface mockup saved as JPEG will have visibly degraded, fuzzy text. Always use PNG or lossless WebP for screenshots.

**Using PNG for photographs.** A product photo saved as PNG will be 4–8× larger than it needs to be. Use JPEG or WebP. The lossless quality of PNG provides no visible benefit for photographs.

**Quantizing images with smooth gradients.** If your PNG has a smooth gradient background, quantization will introduce color banding. Either keep it in full-color PNG, use WebP, or accept the trade-off consciously.

**Forgetting to check transparency before converting.** If you convert a transparent PNG to JPEG, the transparent areas become opaque white. Always check the output format supports transparency if your image has a transparent background.

## FAQ

**Is "compress PNG without losing quality" actually possible?**

Yes, through lossless optimization (recompressing more efficiently) and quantization (reducing the color palette). Lossless optimization saves 10–30% with truly zero quality change. Quantization saves 40–80% with changes that are typically invisible to the human eye but are technically lossy. Converting to lossless WebP saves ~26% vs PNG with zero quality change.

**Why are my PNG files so much larger than JPEGs?**

Because PNG uses lossless compression, which preserves all pixel data. JPEG's lossy compression removes image data you can't see, resulting in dramatically smaller files for photographic content. For a photo, JPEG at 85% quality might be 150 KB while the PNG equivalent is 1.5 MB. The PNG is not "better" for most web use cases — it's just unnecessary.

**Can I preserve transparency when converting PNG to WebP?**

Yes. WebP fully supports alpha channel transparency. When you convert a transparent PNG to WebP, the transparent areas are preserved exactly. The resulting WebP can be placed on any background and will composite correctly, just like the original PNG.

**When should I use PNG instead of WebP?**

Use PNG when: (1) the file needs to work in non-browser applications that don't support WebP (older versions of Word, PowerPoint, some design tools), (2) you need guaranteed pixel-perfect lossless quality with no algorithm differences, or (3) you're producing assets for print workflows. For everything web-delivered in 2026, WebP is a smaller, equivalent-quality alternative.
